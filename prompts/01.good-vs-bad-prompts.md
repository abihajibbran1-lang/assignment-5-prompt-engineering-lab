# Mission 1 — Bad Prompt vs Professional Prompt

## Objective
Compare how prompt quality changes the usefulness, safety, reusability, and testability of an AI-generated JavaScript password-strength checker.

## Issue / Branch / PR
- Issue: `#1 — Compare weak and professional password checker prompts`
- Branch: `doc/issue1-good-bad-prompt`

## Prompt A — Bad Prompt

```text
Make a password checker in JavaScript.
```

### Expected AI Output
Usually a short function that checks only length or a few regex rules. It may return a vague label, omit invalid-input handling, manipulate the DOM, or provide no tests.

### What This Prompt Misses
- No role or intended user
- No clear requirements
- No return format
- No edge cases
- No restriction against DOM manipulation or libraries
- No manual verification request

## Prompt B — Better Prompt

```text
Create a reusable JavaScript function that checks whether a password is weak, medium, or strong. Check length, uppercase letters, lowercase letters, numbers, and special characters. Return the result and feedback. Do not manipulate the DOM. Include examples.
```

### Expected AI Output
A reusable function with several checks and feedback. It should be better than Prompt A but may still leave score thresholds, invalid inputs, whitespace behavior, and output structure ambiguous.

## Prompt C — Professional Prompt

```text
Role: Act as a senior JavaScript mentor reviewing code for a beginner intern.

Task: Create a reusable password-strength checker function named checkPasswordStrength.

Context: The function will be used by a registration form, but it must remain independent of the browser UI so it can be manually tested and reused.

Constraints:
- Use plain JavaScript only and no external libraries.
- Do not access or manipulate the DOM.
- Accept one value named password.
- Handle non-string and empty input safely.
- Check: minimum 8 characters, uppercase, lowercase, number, special character, and no whitespace.
- Use readable names and beginner-friendly logic.
- Do not claim the function guarantees real-world password security.

Output Format:
1. Return code first.
2. Return an object with score, strength, checks, and feedback.
3. Then explain the scoring rules.
4. Then provide at least six manual test cases with expected results.

Edge Cases:
- Empty string
- Non-string input
- Password containing spaces
- Exactly 8 characters
- Missing one required category
- Password satisfying every rule

Verification: Explain how I should run and manually verify the function in Node.js.
```

### Expected AI Output
A pure function with a stable structured return value, transparent checks, invalid-input handling, useful feedback, and test cases. It also added the appropriate package.json where needed and test cases for the strength checking when running npm command.



## Verification Notes
Run `npm test`, then manually call the function with the cases in `tests/manual-test-cases.md`.
