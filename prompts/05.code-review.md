# Mission 5 — Strict AI Code Review

## Issue / Branch / PR
- Issue: `#5 — Review and harden JavaScript utility functions`
- Branch: `doc/code-review`

## Review Prompt

```text
Role: Act as a strict senior JavaScript engineer performing a pre-merge review.

Task: Review the four supplied utility modules: passwordStrength.js, validator.js, discountCalculator.js, and textAnalyzer.js.

Review for:
- correctness and hidden bugs
- naming and readability
- duplicated logic
- validation and edge cases
- security or privacy concerns
- maintainability
- beginner mistakes
- test coverage gaps

Constraints:
- Do not rewrite everything.
- Do not invent product requirements.
- Separate blocking issues from optional suggestions.
- For every finding, cite the file/function and explain a reproducible failure case.
- Prefer minimal changes.
- Reject cosmetic suggestions that do not improve correctness or clarity.

Output Format:
1. Summary verdict: approve, comment, or request changes
2. Findings table: severity, location, problem, reproduction, recommendation
3. Top three fixes to apply now
4. Tests to add
5. Suggestions deliberately not worth applying
```

## Sample Useful AI Findings
1. Reject non-object form input instead of allowing confusing property reads.
2. Prevent multiple validation messages from overwriting each other.
3. Validate discount boundaries and non-finite numbers.
4. Define sentence-count behavior for text without terminal punctuation.
5. Reject password whitespace explicitly.

## Three Suggestions Applied Manually
- Added top-level form-object validation.
- Added `Number.isFinite` and percentage boundaries to the discount calculator.
- Added explicit no-whitespace password feedback.

## Suggestion Not Applied
“Use a complex enterprise validation framework.” This violates the plain-JavaScript constraint and would add unnecessary dependencies.
