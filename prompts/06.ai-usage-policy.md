# Mission 6 — Responsible AI Usage

## Issue / Branch / PR
- Issue: `#6 — Document responsible AI usage policy and report`
- Branch: `doc/ai-policy`


## Policy-Writing Prompt

```text
Role: Act as a software engineering governance reviewer.

Task: Draft a concise responsible-AI usage policy for developers using coding assistants.

Context: Interns may use AI for suggestions, code generation, debugging, refactoring, documentation, and reviews, but they remain accountable for the submitted work.

Required Coverage:
- privacy and personal data
- company/client confidential data
- passwords, API keys, tokens, certificates, and secrets
- .env files and configuration
- copyrighted or licensed code concerns
- manual testing and security verification
- avoiding blind copying
- honest disclosure of AI assistance
- prompt/output/token records
- human ownership and ability to explain the work

Constraints:
- Do not claim AI output is automatically secure or correct.
- Use direct, enforceable language.
- Include a pre-prompt checklist and a pre-merge checklist.
- Keep it suitable for a small software company and interns.

Output Format:
1. Purpose
2. Allowed uses
3. Prohibited data and actions
4. Verification requirements
5. Disclosure and record-keeping
6. Checklists
7. Incident response
```

## Expected Result
A practical policy that explicitly prohibits sharing secrets and confidential data, requires testing, and demands honest documentation.

## AI Mistake to Watch For
Generic policies often say “avoid sensitive data” without naming API keys, `.env` files, client records, private source code, or credentials.
