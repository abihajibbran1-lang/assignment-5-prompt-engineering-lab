## Refactoring Prompt

```text
Role: Act as a senior JavaScript refactoring mentor.

Task: Refactor the supplied text-analysis function for readability and maintainability without adding new user-facing features.

Context: The current function works for simple text but uses unclear names, repeated expressions, var, loose equality, and an inconsistent error shape.

Constraints:
- Plain JavaScript only.
- Preserve the existing analysis categories: characters, characters without whitespace, words, sentences, vowels, and average word length.
- Do not add sentiment analysis, reading time, keyword extraction, or any new feature.
- Use descriptive names, const, strict equality, early validation, and small readable expressions.
- Keep it as one exported reusable function.
- Do not manipulate the DOM.
- State any behavior ambiguity before choosing a rule.

Output Format:
1. Refactored code
2. A before-vs-after table
3. Any behavior ambiguity found
4. Manual regression tests proving behavior was preserved

Edge Cases:
- empty string
- whitespace-only string
- punctuation without words
- multiple spaces/newlines
- non-string input
```

## Expected AI Improvement
Descriptive names, no repeated `trim()`, structured return fields, strict validation, and a clearer sentence-counting expression.

## AI Mistake to Watch For
Refactoring models often quietly change behavior—for example by adding new metrics, removing punctuation handling, or counting decimal points as sentences without documenting the choice.

## Manual Final Version
The final reference implementation keeps the same six metrics, returns a structured error for non-string input, and rounds average word length to two decimals.
